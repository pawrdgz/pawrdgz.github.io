If Found...
UX Case Study

    2020

  • tags

    UI/UX Design, Visual Novel, Videogames

If Found… (2019), by DREAMFEEL studio is a visual novel in which the reader will advance by gradually erasing the screen. You play as Kasio, a trans woman living in the early 1990s in Ireland, a background that serves to downplay issues such as transitioning and social acceptance.

Overview

Although we might think that the erasing system is arbitrary and rather not important, it downplays a metaphor in which the character goes on erasing her past in order to become a new individual. This mechanic is quite easy to understand, yet there are some UI things that could be changed in order to upgrade the experience.

Before we begin, we must acknowledge that the game plays better in its mobile version (tablets and mobile phones), as there are some systems that feel uneasy on PC (e.g. The erasing system itself or the menu navigation, which consists in dragging the screen, both shown in the next images).


Problem

Firstly, the UI is almost non existent, as the only menu that we encounter is the Chapters and Settings. Nonetheless, the player must not do any other action than erasing the screen.

And here we mention the first problem: the mechanic of erasing the screen. It is vaguely shown at the beginning and although it is easily understood on mobile due to the nature of the device, it might be hard to comprehend on PC.


Solution

Since there is no proper call to action -in the PC version- that might tell the user to drag instead of just clicking the screen, I would add an indicator of what the user should do. This is done by simply adding a visual cue in the PC version.

Problem

Throughout the experience, we find some Irish words that are accompanied with a cross (which are actually Irish Celtic Crosses). This indicates that an explanation for the word can be found, although it is rather difficult to understand that it’s a clickable item or that it works as a Glossary.


Solution

To solve the problematic of the player not knowing that it is a clickable item, for the first time that a cross appears, I would add a visual cue. This would lead the user into clicking on the cross at the bottom left of the screen and thus showing how it works.
